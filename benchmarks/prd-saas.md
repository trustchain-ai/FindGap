# PRD / 需求 — Tier-1 标杆

> 行业认可的 SaaS PRD 必含章节 + 量化基线。SKILL.md G4/G9.1/G14.M 扫描会引用本文件。

## 必含章节（缺一即触发 G4）

- [ ] 业务目标（解决什么真实问题，量化预期收益）
- [ ] 用户画像 / Persona（谁会用，使用频次）
- [ ] 验收标准 AC（每条功能 ≥1 条 testable AC）
- [ ] 用户旅程 / 关键场景（happy path + 至少 2 个 error path）
- [ ] 边界条件（极端输入怎么办）
- [ ] 错误处理（出错怎么办、给用户什么反馈）
- [ ] 性能要求（量化指标，见下文 SLO 标杆）
- [ ] 合规要求（隐私 / 安全 / 监管 / 行业特定）
- [ ] 优先级 / 不做什么（out of scope 明确列出）
- [ ] 度量 / 埋点（如何知道这个功能"成功"）

## SLO/SLA 量化基线（G9.1）

### 交互响应

| 场景 | P50 | P95 | P99 |
|------|-----|-----|-----|
| 实时交互（搜索/输入） | < 200ms | < 500ms | < 1s |
| 一般查询 | < 500ms | < 1s | < 2s |
| 批量/后台 | < 5s | < 30s | < 60s |
| 支付下单 | < 300ms | < 800ms | < 2s |

### 可用性

| 场景 | 月度可用性 |
|------|-----------|
| 核心交易 | 99.95%（≤ 22 分钟/月） |
| 普通 SaaS | 99.9%（≤ 44 分钟/月） |
| 非关键内部 | 99.5%（≤ 3.6 小时/月） |

### 成功率

| 场景 | 目标 |
|------|------|
| 支付 / 下单 | ≥ 99.9% |
| 推送送达 | ≥ 99% |
| 文件上传 | ≥ 99.5% |

## 主观词高频位置（G14 重点扫描）

| 章节 | 常见主观词 | SMART 改写方向 |
|------|----------|-------------|
| 性能要求 | "快/流畅/稳定" | P99 + 数字 + 时间窗口 |
| 用户体验 | "友好/简洁/优雅" | N 步内完成 / 学习时长 ≤ X 分钟 |
| 时间 | "尽快/近期/优先" | YYYY-MM-DD 截止 + 里程碑 |
| 范围 | "相关功能/类似/某些" | 枚举具体功能列表 |

## 隐含假设清单（G2 必查）

PRD 通常默认但常出错的假设：

- 用户网络环境（4G/WiFi/弱网占比？）
- 用户设备能力（Android 版本 / iOS 版本下限）
- 数据规模（首日 / 1 月 / 1 年的量级）
- 并发量（峰值 EPS / TPS）
- 国际化（是否多语种、时区、币种）
- 多端一致性（Web / Mobile / Desktop 是否同步）

## 责任链（G8 必查）

- Owner（产品负责人）
- Approver（决策者）
- Reviewer（技术 / 设计 / 法务 / 数据）
- Stakeholders（业务方 / 客户代表）
- 升级路径（决策不下来找谁）

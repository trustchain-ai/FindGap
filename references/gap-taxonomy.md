## G1-G8 基础逻辑层

| Gap | 名称 | 检测信号 | 严重度上限 |
|-----|------|---------|----------|
| G1 | 主观锚词未量化 | 出现"快/好/友好/主流/合理"等无量化词 | 中 |
| G2 | 隐含假设未声明 | 涉及边界但缺"前提是/假设" | 高 |
| G3 | 角色心智漂移 | 同一概念在不同位置/角色描述不一致 | 高 |

| G5 | 来源缺失 | "最佳实践/行业标准/通常"无具体引用 | 中 |

| G7 | 不可证伪 | "提升 X/优化 Y"无可验证标准 | 中 |

## G14 SMART 双向扫描

### G14.S — Specific 具体性

**信号**：形容词式目标 / 范围模糊词 / 动词缺宾语 / 主体不明 / 目标嵌套
**严重度**：critical = 只有形容词无名词宾语 / high = 抽象集合需追问 / medium = 上下文可推断
**改写模板**：
```
在 [具体模块/页面/接口] 上，由 [责任方] 完成 [可指认动作]，
针对 [明确对象] 的 [可定义属性]。
```
**示例**：「我们要优化用户体验」→「前端团队在【订单详情页】的【退款入口】将【点击到结果展示】的【交互步骤】从 5 步减少到 2 步」

### G14.M — Measurable 可量化

**信号**：通篇无数字 / 伪量化词（显著/大幅）/ 缺基线 / 缺口径 / 体感词目标
**严重度**：critical = 无数字且涉及决策投入 / high = 有数字但缺基线 / medium = 缺过程指标
**改写模板**：
```
将 [指标名]（口径：[分子/分母/统计窗口]）从 [基线值] 提升/降低到 [目标值]，
过程指标观察 [二级指标]。
```
**示例**：「显著降低崩溃率」→「Android 客户端【启动崩溃率】（crash 次数 / 启动次数，7 日滑动窗口）从【0.8%】降至【0.2%】以下」

### G14.A — Achievable 可达成

**信号**：目标值断崖 / 无依赖盘点 / 资源失配 / 口号式 / 缺降级方案
**严重度**：critical = 物理不可能或绝对值 100%/0% / high = 缺关键依赖 / medium = 偏激进无 Plan B
**改写模板**：
```
基于 [历史基线/同类项目数据]，在 [资源 X 人 × Y 周 + 预算 Z] 下，
目标 [合理区间]；关键依赖：[依赖 1, 2]；
若 [风险项] 触发，降级到 [Plan B]。
```
**示例**：「下个月把崩溃率降到 0」→「下个月将 Top3 高频崩溃栈修复，崩溃率从 0.8% 降至 0.3%（历史最优 0.25%）；依赖 QA 提供 crash 聚类 + Bugly 接入；若 SDK 侧占比 >50%，降级到 0.5%」

### G14.R — Relevant 相关性

**信号**：无上位目标 / 技术指标未连业务结果 / 为做而做 / 跨目标冲突未化解
**严重度**：critical = 与上位完全无关 / high = 相关性弱或冲突未化解 / medium = 优先级模糊
**改写模板**：
```
本目标支撑 [上位目标/季度 OKR X]，预期对 [核心业务指标 Y] 贡献 [量化收益]；
主要受益方：[角色]；与 [其他目标 Z] 的优先级关系：[高于/低于/并列]。
```

### G14.T — Time-bound 有时限

**信号**："尽快/近期/下一步" / 只有终点无里程碑 / 节点无验收物 / 评审节点缺失
**严重度**：critical = 完全无时间约束 / high = 终点无里程碑且跨度 >2 周 / medium = 验收物模糊
**改写模板**：
```
截止 [YYYY-MM-DD]（自然/工作日），里程碑：
- M1 [日期+交付物+验收标准]
- M2 [日期+交付物+验收标准]
评审节点：[日期+形式]；
外部依赖到位时间：[依赖方+日期]；
缓冲：[N 天]。
```

## 主观词黑名单（G14 触发集）

发现即标 G14.{S|M|A|R|T}，按对应模板改写：

```
好 / 差 / 友好 / 主流 / 合理 / 大致 / 差不多 / 尽快
显著 / 大幅 / 明显 / 适当 / 充分 / 优秀 / 流畅 / 稳定
高效 / 优雅 / 完善 / 灵活 / 强大 / 智能 / 简洁
优化 / 提升 / 增强 / 改善 / 加强 / 健全
```

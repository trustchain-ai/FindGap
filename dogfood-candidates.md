# v0.3 Dogfood 10 候选案例（通用版）

> **使用原则**：照妖镜是**通用 gap 识别协议**，不绑定任何业务领域、行业、角色、技术栈。
>
> 下列 10 个候选是**抽象化案例模板**，演示 G1-G14 的触发场景。实际 dogfood 时把你手头的真实评审产物（PRD / PR / 技术方案 / 会议纪要）丢进去即可，不需要套用本文的具体案例。
>
> **覆盖度检查**：G1-G14 共 14 类 gap，每类至少被 1 个候选触发。详见末尾对照表。

---

## 候选清单

### 1. [PRD] 某 B2B SaaS · 多租户隔离方案

- **选材理由**：典型"功能描述清晰但隐含假设不明"场景。会暴露：
  - G1 主观锚词（"安全""高性能"无量化）
  - G2 隐含假设（租户规模假设、数据隔离粒度假设、合规框架假设）
  - G12.1/G12.3/G12.4 监管合规（DSR / 跨境 / 保留期）
  - G14.M/G14.S 缺基线 + 不具体
- **预期严重度分布**：P0×2 / P1×3 / P2×2
- **难度**：medium

### 2. [PRD] 某数据基础设施 · 实时计算链路 PRD

- **选材理由**：典型"性能 SLO 静默"场景。会暴露：
  - G4 未问的关键问题（无延迟/吞吐量/数据规模指标）
  - G9.1 SLO/SLA 量化基线缺失
  - G9.3 可观测性三件套不完整
  - G9.7 容量评估缺失
  - G14.M/G14.T 缺数字 + 缺日期
- **预期严重度分布**：P0×2 / P1×4 / P2×1
- **难度**：medium

### 3. [PRD] 某客户端 · 离线优先功能 PRD

- **选材理由**：典型"边界条件缺位"场景。会暴露：
  - G4/G6 错误处理与反例缺失（弱网/断网/重连未考虑）
  - G9.5 弱网/离线/重连场景缺失
  - G9.2 幂等设计缺失
  - G11.4 核电纵深防御缺失（多层 fallback）
- **预期严重度分布**：P0×3 / P1×2 / P2×1
- **难度**：high

---

### 4. [PR] 某接口空值兼容补丁

- **选材理由**：典型"小补丁但影响面未评估"场景。会暴露：
  - G2 隐含假设（默认上游一定返回某字段）
  - G3 角色心智漂移（调用方/被调用方对空值约定不一致）
  - G6 反例缺失（极端输入未测）
  - G5 来源缺失（基于哪个版本契约写的）
- **预期严重度分布**：P0×0 / P1×2 / P2×2 / P3×1
- **难度**：low

### 5. [PR] 某跨模块状态机重构 PR

- **选材理由**：典型"大范围改动 + 责任真空"场景。会暴露：
  - G8 责任真空（多模块改动谁 review 谁负责未明）
  - G3 角色心智漂移（不同模块对状态定义不一致）
  - G10.5 迁移/上线类历史事故风险（无双写校验/影子读）
  - G9.4 灰度策略未明
- **预期严重度分布**：P0×1 / P1×3 / P2×2
- **难度**：high

### 6. [PR] 某依赖升级 + 安全补丁 PR

- **选材理由**：典型"安全相关但流程不完整"场景。会暴露：
  - G10.1 CVE 同类漏洞复现（升级是否真覆盖 CVE）
  - G10.6 供应链投毒未防护（无 lockfile / 无 SBOM）
  - G12.10 开源许可证盲区
  - G14.M 量化缺失（升级前后性能/兼容性对比无数字）
- **预期严重度分布**：P0×1 / P1×2 / P2×2 / P3×1
- **难度**：medium

---

### 7. [技术方案] 某分布式系统 · 跨地域路由表同步方案

- **选材理由**：典型"跨系统一致性 + 反例缺失"场景。会暴露：
  - G6 反例缺失（路由表新旧混跑未论证）
  - G9.8 数据一致性策略未明示（Saga/TCC/补偿）
  - G10.2 知名 postmortem 风险（重试风暴/反向 DNS）
  - G11.1 金融对账思路缺失（独立校验）
  - G9.6 限流/熔断缺失
- **预期严重度分布**：P0×2 / P1×3 / P2×1
- **难度**：high

### 8. [技术方案] 某代码生成约束规则的工程化落地方案

- **选材理由**：典型"规则定义清晰但执行机制模糊"场景。会暴露：
  - G7 不可证伪（"提升代码质量"无可验证标准）
  - G14.A 可达成性（绝对化目标"100% 命中"）
  - G14.R 相关性（与上位目标关系未说明）
  - G13.7 现状偏差（"我们一直这么做"无重评估）
- **预期严重度分布**：P0×0 / P1×3 / P2×3 / P3×1
- **难度**：medium

### 9. [对话/会议纪要] 某跨部门 (产品/技术/测试/合规) 拉通会纪要

- **选材理由**：典型"多方共识但语义漂移"场景。会暴露：
  - G3 角色心智漂移（同一术语不同角色理解不同）
  - G8 责任真空（拉通了但无人 owner）
  - G2 隐含假设（多方各自带未声明前提）
  - G11.2 医疗 Checklist 缺失（无 time-out 核对动作）
  - G12.x 合规视角集体盲区
- **预期严重度分布**：P0×1 / P1×3 / P2×3
- **难度**：medium

### 10. [对话/会议纪要] 某重大发版决策评审纪要

- **选材理由**：典型"集体决策但反例未论证"场景。会暴露：
  - G6 反例缺失（无 pre-mortem）
  - G13.2 确认偏误（支持证据 >> 反对证据）
  - G13.3 规划谬误（估时低于历史中位数）
  - G13.5 乐观偏差（"应该没问题"无量化依据）
  - G11.6 心理学 Pre-mortem 缺失
  - G9.4 灰度策略 / 回滚预案
- **预期严重度分布**：P0×2 / P1×3 / P2×2
- **难度**：high

## 使用方法

1. **不要照搬本文的具体业务描述**——它们只是触发模式的示意。
2. **拿你真实手头的评审产物**（PRD / PR diff / 方案文档 / 会议纪要）丢给照妖镜。
3. **dogfood 目标 10 个产物 + 4 类分布**（PRD×3 / PR×3 / 方案×2 / 对话×2）。
4. 记录每次扫描的输出，统计 P0/P1/P2/P3 分布 + 是否触发了"未问的关键问题"类深度 gap。
5. 用反馈钩子（`/zhaoyaojing-feedback`）回填 accepted/rejected/known/fix-applied。

## v0.1 — 草稿（已完成）

**核心范围**
- 定义"反身性审视"骨架：触发词识别、单轮 gap 列表输出
- 最小工件：一份 prompt + 一个触发器表
- 仅在本地 IDE / Claude Code 内运行

**工期**：2 周（已结束）

**退出条件**
- 能识别"照一下/审一下/有没有遗漏"等核心触发短语
- 能输出 ≥3 条 gap 的非空列表
- 用户能在 1 轮交互内拿到首版反馈

**验收**
- 在 10 个内部 PRD / PR 评审场景中触发成功率 ≥ 80%
- 至少 1 名核心用户每周主动调用 ≥ 3 次

## v0.2 — Gap 识别能力大扩展（已发布）

**核心范围**
- gap 分类体系成型：事实 / 逻辑 / 范围 / 风险 / 盲区 5 类
- 多轮收敛：从"列 gap"升级为"列 gap → 排优先级 → 推荐消除手段 → 对齐"
- 触发场景扩展：PRD 评审、PR 评审、技术方案、对话决策、复盘
- 输出结构化：每条 gap 带 类别 / 证据 / 建议动作 / 置信度

**工期**：4 周（已结束）

**退出条件**
- 5 类 gap 分类在 prompt 中固化并稳定输出
- 多轮收敛 loop 默认开启，平均 ≤ 3 轮达成"对齐"
- 至少覆盖 5 类协作场景的触发模板

**验收**
- 内部 30+ 真实案例中，用户接受的 gap 占比 ≥ 60%
- 误报（用户标记"无效 gap"）比例 ≤ 25%
- 多轮收敛成功率（最终用户确认"已对齐"）≥ 70%

## v0.3 — 双轨报告模板 + dogfood 闭环（当前）

**核心范围**
- **双轨报告模板**：
  - 轨道 A — 协作者视角：给被审视的人，强调"你下一步该做什么"
  - 轨道 B — 反身视角：给审视的发起人，强调"你自己的预设有哪些被照出来"
  - 两轨在同一次调用中并列产出，互为镜像
- **dogfood 闭环**：
  - 每次调用结果回收：用户标注 gap 是否有效 / 是否触达盲区
  - 标注数据回流到 prompt 迭代，形成"用 → 反馈 → 改 prompt → 再用"周期
  - 维护一个 dogfood 记录表（场景 / 调用 / 接受率 / 改进点）

**工期**：6 周（进行中，当前为 Week 3）

**退出条件**
- 双轨模板在所有触发场景中默认输出
- dogfood 反馈回收机制可用（每次会话末尾的轻量打分 + 可选标注）
- 至少完成 1 轮基于回收数据的 prompt 迭代

**验收**
- 双轨同时被用户认为"有用"的比例 ≥ 50%
- dogfood 数据样本量 ≥ 80 次调用
- 基于回收数据的迭代，使 gap 接受率较 v0.2 提升 ≥ 10 个百分点

## v0.5 — 数据驱动人格 + 单平台轻接入（计划）

**核心范围**
- **数据驱动人格**：
  - 基于 v0.3 dogfood 数据，刻画"用户类型 × 场景"的差异化审视风格
  - 落地 2 个人格（先聚焦"产品评审"和"技术方案"两类高频场景，不再追求 5 个）
  - 人格切换由触发器自动判定，不需要用户显式选择
- **单平台轻接入**：
  - 选定一个外部平台做轻量集成（候选：飞书 / Notion / GitHub PR 评论之一）
  - 只做"调用入口 + 结果回写"，不做深度嵌入
  - 单平台先跑通闭环，再考虑横向扩展

**工期**：8 周

**退出条件**
- 2 个人格在 prompt 层稳定区分，且自动路由准确率 ≥ 80%
- 单平台接入完成：从平台原生界面可触发 → 结果回写到原上下文
- 接入平台上至少 5 名外部用户活跃使用

**验收**
- 人格差异化带来的 gap 接受率提升 ≥ 8 个百分点（vs v0.3 基线）
- 单平台月活调用 ≥ 200 次
- 外部用户留存（次月仍调用）≥ 30%

## v1.0 — 公开 diff + 白皮书 + 协议化（计划）

**核心范围**
- **公开 diff**：
  - 把 v0.1 → v1.0 的 prompt / 模板 / 决策演进全部公开，作为方法论资产
  - 每个版本附"为什么这样改"的决策记录
- **白皮书**：
  - 替代原计划的 arXiv 论文，定位为方法论白皮书（而非学术论文）
  - 内容：反身性审视的概念、双轨模型、人格机制、dogfood 数据
  - 同步配套：传播渠道合并到 v1.0 一次性铺开（替代原 v0.6 的 HN 双线计划）
- **协议化**：
  - 定义"反身性审视调用"的轻量协议（输入 / 输出 / 反馈回路 schema）
  - 让其他 AI 工具 / 平台可以遵循该协议接入
- **开源许可**：
  - 采用 MIT + 商标保护（替代原计划的 BUSL）
  - 商标仅约束"照妖镜 / zhaoyaojing"品牌使用，代码本身完全开放

**工期**：12 周

**退出条件**
- 公开仓库：包含 prompt diff 历史、白皮书、协议草案
- 至少 1 个第三方工具按协议接入并跑通端到端
- MIT + 商标条款落地，法律审阅通过

**验收**
- 白皮书发布 30 天内：阅读量 ≥ 5k，外部引用 ≥ 10 次
- 协议被 ≥ 2 个独立项目实现
- 终极目标信号：在不直接由作者触发的协作场景中，用户/AI 自发调用"照一下"≥ 50 次/月

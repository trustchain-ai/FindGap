# FindGap · ROADMAP

> 北极星：按 Enter 前找 gap → 有行业数据用行业数据，没有就上下文推理兜底 → 给优化方案 → 用户自己判断
>
> Find* 系列定位：FindGap（事前·找缺口）→ FindAction（事中·找动作）→ FindMiss（事后·找遗漏）。本仓库只覆盖 FindGap。

## v0.7 — 20 次真实闭环 · 建立精度基线（进行中）

**为什么做 v0.7**：v0.6 三节点从未在真实 prompt 上跑出精度数据。没有基线，"调精度"和"找外部用户"都无靶。

**核心交付**（4 件，不多不少）：

1. **判定协议** `dogfood/protocol.md`
   - 真阳定义：命中项确实指出了 prompt 中的含混，且该含混会导致下游理解偏差
   - 假阳定义：命中项指出的问题在上下文中不构成含混（如示例文本、引用、描述性使用）
   - 判定者：prompt 作者本人（v0.7 阶段 = 项目作者）
   - 样本量：≥ 20 条不同 prompt
   - 统计口径：按规则逐条统计 + 总量聚合

2. **20 条真实 `/照` 记录** `dogfood/runs/005-024`
   - 来源：10 条日常工作中的真实 prompt + 10 条从公开场景（GitHub Issues / Stack Overflow / 社区提问）收集
   - 格式：v0.6 三节点（照→查→亮）完整闭环
   - 每条标注：命中项 × 真阳/假阳 + 判定理由

3. **基线报告** `dogfood/baseline.md`
   - 11 招各自的：命中次数 / 真阳数 / 假阳数 / 精度（precision）
   - 总量聚合：总 FPR / 总 TPR
   - 分层发现：哪几条招精度高（可信赖）、哪几条需要调（v0.8 靶子）

4. **ROADMAP 回填**
   - 用基线数据修正 v0.8 退出条件（FPR/TPR 目标基于实际数据而非空想）
   - 解决 v1.0 scope 冲突（本版本决策并记录 ADR）

**不做**（推后）：
- ❌ 找外部用户（v0.8，等基线证明可用后再推）
- ❌ 调规则精度（v0.8，等知道哪条需要调）
- ❌ WebSearch 模板优化（v0.8，跟着精度走）
- ❌ 方法论白皮书（v1.0）
- ❌ 查 PRD/PR/对话记录（v1.0 决策项，本版 ADR 记录）

**退出条件**（4 项全 true 才算完）：

```bash
# 1. 判定协议存在
test -f dogfood/protocol.md && echo PASS

# 2. ≥ 20 条 v0.6 格式记录
find dogfood/runs/ -name "*.md" -newer dogfood/runs/004-self-v05-grayfirst.md | wc -l  # ≥ 20

# 3. 基线报告含 11 条规则各自精度
grep -cE "S-PERF|S-QUANT|S-NFR|R-DOD|G-WHY|G-NOGO|I-SSOT|I-ADR|V-NAME|V-STAKE|V-LAYER" dogfood/baseline.md  # ≥ 11

# 4. ROADMAP v0.8/v1.0 各有退出条件
grep -c "退出条件" ROADMAP.md  # ≥ 3
```

**工期**：5 天（1 天写协议 + 3 天跑 20 条 + 1 天出报告回填）

## v0.8 — 测量校准优先 + 外部灰度验证 + 高杠杆规则增强（计划）

**启动条件**：`dogfood/baseline.md` 已产出，且 20 条基线已确认

**为什么做 v0.8**：v0.7 已证明检测器整体可用，但当前证据仍以 precision 为主，TPR 缺分母，判定者也主要是 prompt 作者本人。11 条规则里 9 条 precision = 1.0000，总 FP 仅 3，说明 v0.8 不能再平均用力；但在强化高 ROI 规则前，必须先把“测量是否可靠、命中后是否真能改 prompt”校准清楚。

**v0.8 方向**：
- 先校准测量：补独立判定、rewrite adoption、coverage / recall proxy、样本分层
- 再做外部灰度：验证“命中后是否有用、是否真能改 prompt”
- 最后强化高杠杆规则：`R-DOD`、`V-STAKE`、`V-NAME`、`I-SSOT`、`I-ADR`、`G-NOGO`
- 低精度规则仍做定点调优：先 `V-LAYER`，后 `G-WHY`

**核心范围**：
1. **测量校准**
   - 在外部样本中增加独立判定者，不再只依赖 prompt 作者本人
   - 为每条记录新增“是否据此改写 prompt / 是否采纳命中项”结果字段
   - 为样本增加类型分层（如 PRD / 技术方案 / 一句话需求），避免频次结论被单一领域绑架
   - 建立 recall / coverage proxy，不再只看 precision
2. **外部灰度验证**
   - 首批外部用户不只验证“命中没命中”，而验证“命中后是否真改了 prompt，是否减少了分歧”
3. **高杠杆规则增强**
   - `R-DOD`：强制给出可观察输出、完成条件、可测试通过标准
   - `V-STAKE`：强制写清谁受影响、错了会损失什么、为什么现在做
   - `V-NAME`：要求规范名 / 别名对照
   - `I-SSOT`：要求权威来源链接或明确“暂无 SSOT”
   - `I-ADR`：要求最小决策理由
   - `G-NOGO`：要求明确不做项与边界
4. **定点调优低精度规则**
   - 收窄 `V-LAYER` 判定条件，减少“业务层与实现层并列描述但未造成歧义”的误报
   - 收窄 `G-WHY` 判定条件，减少“stakes 或 DoD 已隐含 why”时的过判

**绝对目标**（全部基于 v0.7 真实基线 20 runs / 154 hits / 151 TP / 3 FP）：
- 总 FP：从 3 降到 ≤ 2（需在 matched-strata 且新样本 ≥ 20 条下验证，不以单池绝对数字为准）
- `V-LAYER`：FP rate 在新样本中低于 v0.7 基线（2/13 = 15.4%），不设绝对零目标
- `G-WHY`：FP rate 在新样本中低于 v0.7 基线（1/13 = 7.7%），不设绝对零目标
- `R-DOD`、`V-STAKE`、`V-NAME`、`I-SSOT`、`I-ADR`、`G-NOGO`：新样本中 FP rate 不回归（即不高于 v0.7 基线的 0%）
- 外部灰度样本：≥ 9 条有效 E2 记录
- 每条外部记录新增”是否改写 prompt / 是否采纳命中项”字段
- 新样本需按类型分层，不再只做单池统计
- **统计有效性约束**：上述目标在新样本量 < 20 条时不做硬判定，仅作方向性参考

**退出条件**：
- 新一轮样本（≥ 20 条）中总 FP rate 低于 v0.7 基线
- `V-LAYER` 新样本 FP rate < 15.4%，`G-WHY` 新样本 FP rate < 7.7%
- 上述 6 条高杠杆规则新样本 FP rate 仍为 0%
- 外部用户累计 ≥ 9 条有效 E2 记录，且记录中包含”命中后是否改写 prompt / 是否采纳命中项”的结果字段
- 至少有一轮独立判定（非 prompt 作者本人）结果可回看
- 有可运行的 recall / coverage proxy 口径
- ROADMAP、基线报告、规则文案三者已按同一口径回填

**不做**：
- 不重写整套 11 招
- 不把 `S-PERF / S-NFR` 升为主战场
- 不在缺数据时预设主观满意度阈值
- 不扩 scope 到 PRD / PR / 对话记录

## v1.0 — 11 招精度稳定 + 跨模型验证（计划）

**启动条件**：v0.8 退出条件全部达成

**为什么做 v1.0**：11 招规则在多个模型上精度稳定，可以作为 AI 协作标准工具推荐给社区。

**核心范围**：
- 11 招规则冻结（连续 2 个版本无精度回归 = 冻结条件）
- 跨模型验证：在 ≥ 3 个不同 LLM 上（Claude / GPT / Gemini）跑 10 条相同输入，精度偏差 ≤ 15%
- 方法论白皮书：总结 v0.1-v1.0 的砍/留决策 + 行业数据汇总

**退出条件**：
- 11 招连续 2 版本精度无回归（各招 precision 波动 ≤ 5%）
- 跨 ≥ 3 模型精度偏差 ≤ 15%
- 累计真实 `/照` 记录 ≥ 100 条（含外部用户）
- 白皮书已发布

**ADR：v1.0 不做"查 PRD/PR/对话记录"**
> 决策日期：2026-06-03
> 背景：SKILL.md 杜绝列表第 4 条写"查 PRD/PR/对话记录（那是 v1.0 的事）"。本次自照（I-SSOT + G-NOGO）发现这与 ROADMAP v1.0 scope 冲突。
> 决策：v1.0 scope 限定为"11 招精度稳定 + 跨模型验证"。扩展照射范围（PRD/PR/对话）属于 v2.0+ 的独立产品方向决策，不纳入当前路线。
> 理由：FindGap 的北极星是"按 Enter 前找 gap"，扩展到 PRD/PR/对话改变了产品定位——需要独立立项论证，不能塞进 v1.0。
> 备选方案（不采用）：把"查 PRD/PR"作为 v1.0 必选项——被否决，因为会打破"11 招冻结"的稳定性目标。

## 词汇表

| 术语 | 定义 |
|------|------|
| 真阳 (TP) | 命中项确实指出了输入中的含混 |
| 假阳 (FP) | 命中项在上下文中不构成含混 |
| FPR | 假阳性率 = FP / (FP + TP) |
| TPR | 真阳性率 = TP / 应检出总数（需人工标注全量 gap 后回算）|
| precision | 精度 = TP / (TP + FP)，每条招独立计算 |

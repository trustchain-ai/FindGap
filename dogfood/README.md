# Dogfood · 真实数据回收目录

> **目的**：照妖镜作为自己的第一灰度用户，将每次"自我审视"产出的真实数据沉淀于此，驱动版本迭代。
>
> **学术对照**：本目录的存在是对 v0.3 ROADMAP "dogfood 数据回收中"的实证回应——在 v0.4 启动日之前，全部为 0 条真实记录（v0.3 自照报告已暴露）。

---

## 目录结构

```
dogfood/
├── README.md              本文件
├── runs/                  每次自照/外照的原始记录
│   └── NNN-{slug}.md      按编号顺序，slug 描述场景
├── feedback.jsonl         汇总反馈（追加写，权威来源）
├── stats.md               周报式聚合统计（手工更新）
└── conventions.md         数据采集规范
```

## 记录规范

每条 `runs/NNN-*.md` 必须包含：

1. **元数据头**：scanId、产物路径、产物版本、扫描时间、扫描者（self / external / agent）
2. **输入快照**：被扫描产物的引用（git sha + 文件路径），不重复全文
3. **门禁结果**（v0.4 起）：readinessScore + verdict + 阻断/警告/建议数
4. **完整 finding 列表**：含 ID、严重度、定位、证据、建议
5. **反馈结论**：accepted / rejected / known / fix-applied + 理由
6. **修复轨迹**：如有修复，记录修复 PR / commit / 修改后复扫的新 ID

## v0.4 灰度策略

照妖镜把"本项目自身的每次迭代"作为第一批真实 dogfood 数据来源：

| 触发 | 自动产生 dogfood 记录 |
|------|---------------------|
| 每次提交 PR/Commit 前 | 跑一次门禁 → 记录到 runs/ |
| 每次发布版本前 | 全量自照 → 记录到 runs/ |
| 用户提交外部产物 | 经同意后脱敏记录到 runs/external/ |

**红线**：dogfood 数据严格服务于本项目迭代，**不**回流进任何在线决策（与 SKILL.md 红线 8 + v0.3 红线 12 一致）。

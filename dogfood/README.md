# Dogfood · 真实数据回收目录

> **目的**：照妖镜作为自己的用户，将每次"自我审视"产出的真实数据沉淀于此，驱动版本迭代。

---

## 目录结构

```
dogfood/
├── README.md              本文件
├── protocol.md            判定协议（v0.7 新增）
├── runs/                  每次自照/外照的原始记录
│   └── NNN-{slug}.md      按编号顺序，slug 描述场景
├── feedback.jsonl         历史反馈汇总（v0.3-v0.4 时期，归档保留）
├── stats.md               历史聚合统计（v0.3-v0.4 时期，归档保留）
└── conventions.md         数据采集规范
```

## 记录规范

每条 `runs/NNN-*.md` 必须包含：

1. **元数据头**：scanId、产物路径、产物版本、扫描时间
2. **输入快照**：被扫描产物的引用（git sha + 文件路径）
3. **三节点结果**：节点 ① 照命中了哪些招、节点 ② 查找到了哪些来源、节点 ③ 亮的完整输出
4. **反馈**：哪些命中有用、哪些是误报

## v0.8 分层灰度说明

- **E1**：内部操作者评公开提示词。回答“这套规则是否可判定、标注是否一致”
- **E2**：真实外部用户使用自己的提示词。回答“命中后是否有用、是否真会改 prompt”

E1 负责 correctness 的可判定性；E2 才开始收集 usefulness signals。两者不得混为一个池统计。

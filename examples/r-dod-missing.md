# Example · R-DOD · 验收标准（DoD）缺失

> **来源**：本仓真实历史命中（v0.4 时期，使用当时的 PASS/FAIL 方法论）。
> 原案例来自 dogfood/runs/001-self-v03-bootstrap.md Finding #4。
> v0.6 已砍掉 PASS/FAIL，输出格式改为三节点（照→查→亮）。下方保留历史原貌。

---

## 场景

照妖镜 v0.3 ROADMAP.md 多处声明：

```
状态：v0.3 机制铺设里程碑（v0.3.1 修复中），dogfood 数据回收中
```

```
退出条件：
- dogfood 数据样本量 ≥ 80 次调用
```

但仓内当时（v0.3 发布时刻）**真实 dogfood 记录为 0**，"回收中"无任何完成条件、无时间表、无量化阈值。

---

## 照妖镜输出

```
[FAIL] R-DOD 验收标准（DoD）缺失
  file: ROADMAP.md:4
  原形: "状态：v0.3 机制铺设完成，dogfood 数据回收中"
  why : "回收中"无验收标准——回收多少算够？什么时候算完？
        实际仓内 0 条真实记录，"回收中"等价于不可证伪
        下游（v0.4 规划者）无法判断该里程碑是否真的完成
  补齐: 改为可证伪的 DoD，例如：
        "v0.3 完成 = 仓内存在 ≥10 条真实 dogfood 记录（dogfood/runs/*.md）
         且每条含 ≥1 个真实命中 finding，
         feedback.jsonl 累计 ≥30 条用户反馈"
```

---

## 真实修复

v0.4 commit `2061e03` 已建立 `dogfood/` 目录 + 首条真实记录 R-20260602-001 + feedback.jsonl 18 条入仓。同时 ROADMAP.md 重写为含量化退出条件的 v0.4/v0.5 表（每个版本 4 个 grep 可验指标）。

修复后**同一段重照不再 FAIL**。

---

## 给上游的认知盲点

**"完成中 / 进行中"是最危险的状态词**——它假装是状态，实际是逃避验收。
没有可证伪的完成标准，里程碑就是无限延期。

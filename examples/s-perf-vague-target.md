# Example · S-PERF · 性能/时效形容词无指标

> **来源**：本仓真实历史命中。本案例来自照妖镜自己 v0.4 时期的 ROADMAP.md 终极目标自照（dogfood/runs/001-self-v03-bootstrap.md Finding #2）。
>
> 杜绝虚构：所有原文、行号、修复都可在 git history 查到。

---

## 场景

照妖镜 v0.3 ROADMAP.md 第 5 行写道：

```
终极目标：让"照一下"成为人类与 AI 协作中被默认调用的反身性环节。
```

提交给"下游"（即未来的实施者/外部贡献者）时，被 v0.4 Phase 0 自照拦下。

---

## 照妖镜输出

```
[FAIL] S-PERF 性能/时效形容词无指标
  file: ROADMAP.md:5
  原形: "让'照一下'成为人类与 AI 协作中被默认调用的反身性环节"
  why : "默认调用"是主观词——多频算默认？哪些场景算"协作"？无基线、无分母、无截止
        实施者无法判断目标何时算达成，规划必然分歧
  补齐: 改为可量化的逼近指标，例如：
        "到 2027-12-31，在 ≥3 个独立团队的 PRD/PR/技术方案评审中，
         照妖镜作为执行前置门禁被默认启用；衡量信号：
         - 入 agent 前 PASS 比例 ≥ 70%（基线 v0.3 末估算 30%）
         - 累计真实 dogfood 反馈 ≥ 500 条
         - 外部独立调用方 ≥ 10 个"
```

---

## 真实修复

v0.4 commit `2061e03` 已落地这次补齐——ROADMAP.md 终极目标段重写为 5 指标 SMART 版（含 2027-12-31 deadline + v0.4/v0.5/v1.0 阶段目标表）。

修复后**同一行重照不再 FAIL**。这是 S-PERF 规则在自身灰度中的**第一条真阳样本**。

---

## 给上游的认知盲点

**"默认调用"听起来很清楚**——但任何主观词写入战略目标，下游一定会用不同数字解读。
照妖镜的价值不是替你写数字，是**让你意识到自己没写**。

# 案例 1：评审一份模糊 PRD

## 输入产物

```
PRD：海外 POS 支持 BNPL

业务描述：
- 海外 POS 也要支持 BNPL（先买后付）
- 类似现有扫码支付，加一个新按钮
- 性能要好，体验要流畅
- 上线时间：尽快
```

## 预期照妖镜运行结果

### Phase 1: Detect 识别结果

| # | Gap | 类型 | 严重度 | 命中位置 | 命中证据 |
|---|-----|------|-------|---------|---------|
| 1 | G1 主观锚词未量化 | 中 | 🟡 | "性能要好"、"体验要流畅"、"尽快" | 3 个主观词无量化 |
| 2 | G2 隐含假设未声明 | 高 | 🔴 | "类似现有扫码支付" | 未说明 BNPL 与扫码差异 |
| 3 | G3 角色心智漂移 | 高 | 🔴 | "BNPL" 一词 | 客户/产品/研发对 BNPL 流程的理解可能差异巨大 |
| 4 | G4 未问的关键问题 | 高 | 🔴 | 全文 | 缺失：错误处理、回滚、并发、合规、对账 |
| 5 | G5 来源缺失 | 中 | 🟡 | "类似现有扫码支付" | 未引用现有扫码的具体规范 |
| 6 | G6 反例缺失 | 中 | 🟡 | 全文 | 未讨论失败场景（额度不足、风控拦截、退款） |
| 7 | G7 不可证伪 | 中 | 🟡 | "体验要流畅" | 无验收标准 |
| 8 | G8 责任真空 | 高 | 🔴 | "上线时间：尽快" | 未指定 owner、决策权 |

**初始 severity_score** = 4 高 × 3 + 4 中 × 1 = **16**

### Phase 2: Decide 决策

| Gap | 推荐手段 | 选择理由 |
|-----|---------|---------|
| G1 | 直接给 3 档量化候选 | 中严重度，规则可解 |
| G2 | 列常见前提清单 + web search 同类场景 | 高严重度，BNPL 行业有成熟规范 |
| G3 | 四方心智对比表 | 高严重度，POS 业务跨多角色 |
| G4 | PRD 领域 checklist 自动补全 | 高严重度，PRD 缺核心章节 |
| G5 | 强制要求带 URL（标注或 web search 补） | 中严重度 |
| G6 | 生成 3 类反例（边界/冲突/缺位） | 中严重度 |
| G7 | 把承诺转量化 + 列验证方法 | 中严重度 |
| G8 | 强制 RACI 表 + 明确升级路径 | 高严重度，无 owner 上线必翻车 |

### Phase 3: Eliminate 消除执行

#### G1 量化产出
- "性能要好" → 推荐 BNPL 接口 99% < 500ms（参考 Visa/MC online auth SLO）
- "体验要流畅" → 推荐 BNPL 首次申请流程 ≤ 3 屏，已通过用户 ≤ 1 屏
- "尽快" → 推荐 T+30 天上线（含合规审核），首版只覆盖 1 个市场 1 个 BNPL 商家

#### G4 缺失章节补齐
```
你的 PRD 缺以下章节，建议补齐：
□ 用户故事（哪类商户、哪类持卡人）
□ BNPL 申请流程（首次 vs 后续）
□ 错误处理（额度不足/风控拒绝/服务不可用）
□ 退款流程（BNPL 特殊性：分期未付的部分如何退）
□ 对账（BNPL 提供方 vs 商户 vs POS 后台 三方对账）
□ 合规（当地金融监管要求、消费者保护）
□ 灰度策略（哪个商户先试，多久全量）
□ 验收标准（怎么算成功上线）
```

#### G6 反例
- 边界：用户额度刚好不够 1 分钱
- 冲突：BNPL 申请通过但 POS 网络断了
- 缺位：未约定退款时分期未付部分由谁承担

#### G8 RACI
```
- Responsible（执行）：[未指定] 必填
- Accountable（决策）：[未指定] 必填
- Consulted（咨询）：合规、风控
- Informed（知会）：客服、运维
- 升级路径：技术问题 → 技术 leader；商务问题 → 产品 owner
```

### Phase 4: Converge 收敛

**第 1 轮结果**：
- 初始 gap 数：8（高 4 / 中 4）
- 消除后残留：2（中 2，G5 和 G6 部分残留）
- severity_score：16 → 2

**判定**：✅ severity_score ≤ 2，接近对齐，结束。

---

## 最终对齐报告

**对齐前**：8 个 gap，severity_score = 16，**直接进开发会翻车**
**对齐后**：2 个 gap，severity_score = 2，**可进技术方案阶段**
**对齐率**：87.5%

**残留 gap**（不阻塞但建议处理）：
- G5：BNPL 行业标准引用待补
- G6：极端反例可在技术方案阶段补充

**建议下一步**：
- 把消除后的 PRD 草稿发产品 owner 评审
- 评审通过后进入技术方案阶段（届时可再过一遍照妖镜）
